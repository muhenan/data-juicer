# 去重算法原理

## 为什么需要去重

SFT 数据中大量存在"同一道题被不同人改写成多个版本"的情况，模型反复学习相似数据会导致过拟合，且浪费训练资源。去重是数据清洗的关键步骤。

---

## 一、精确去重（MD5）

**对应算子：`document_deduplicator`**

### 原理

对每条文本计算 MD5 哈希值，哈希值完全相同则视为重复，保留第一条，丢弃后续。

可选参数：
- `lowercase`：比较前先转小写，忽略大小写差异
- `ignore_non_character`：忽略空格、数字、标点，只比较字母内容

### 复杂度

| 步骤 | 复杂度 |
|---|---|
| 计算每条文本的 MD5 | O(L)，L 为文本长度 |
| 哈希表查重 | O(1) 每条 |
| **总复杂度** | **O(nL)**，线性，极快 |

### 局限

只能识别**完全相同**的文本，改了一个字就识别不了。

---

## 二、模糊去重（MinHash + LSH）

**对应算子：`document_minhash_deduplicator`**

### 第一步：文本 → n-gram 集合

把文本按滑动窗口切成 n-gram：

```
"光合作用是植物"（window_size=5）
→ {"光合作用是", "合作用是植", "作用是植物", ...}
```

两条文本越相似，n-gram 集合重叠越多。用 Jaccard 相似度衡量重叠程度：

```
Jaccard(A, B) = |A ∩ B| / |A ∪ B|
```

### 第二步：n-gram 集合 → MinHash 签名

直接比较两个巨大的 n-gram 集合成本太高。MinHash 的做法：

用 `num_permutations` 个随机哈希函数，每个函数从集合中取哈希值最小的元素作为"代表词"，组成固定长度的签名向量。

**一个哈希函数 = 用一种规则从集合里选出一个代表词。**

数学上可证明：两个签名中相同位置值相等的比例 ≈ Jaccard 相似度。

举例（简化为 3 个哈希函数）：

```
集合 A = {"光合作用", "合作用是", "作用是植"}
集合 B = {"光合作用", "合作用是", "用是植物"}

         h1   h2   h3
A的签名: [1,   2,   1]
B的签名: [3,   2,   1]

相同位置数 / 总位置数 = 2/3 ≈ 0.67（真实 Jaccard = 0.5）
```

`num_permutations` 越大，估算越准，计算量也越大。

### 第三步：LSH 避免两两比较

有了签名后，如果仍然两两比较，复杂度还是 O(n²)。**LSH（局部敏感哈希）解决这个问题。**

做法：把签名切成若干个 band，每个 band 单独哈希到一个桶里。**只要有任意一个 band 落入同一个桶，就成为候选重复对。**

```
签名（12位）分成 4 个 band，每个 band 3 行：

文本A: [3,6,1 | 8,2,9 | 4,7,2 | 5,1,3]
文本B: [3,6,1 | 8,2,5 | 4,7,2 | 5,1,3]
        band1   band2   band3   band4
        相同     不同    相同    相同
```

A 和 B 在 3 个 band 中相同 → 进入候选集 → 精确比较确认是否重复。

相似度越高，进入候选集的概率越高；相似度低的对大概率不在同一个桶里，无需比较。

### 复杂度

| 步骤 | 复杂度 |
|---|---|
| 计算 MinHash 签名（k 个哈希函数） | O(kL) 每条 |
| LSH 分桶 | O(k) 每条 |
| 候选对精确比较 | O(c·k)，c 为候选对数量，远小于 n² |
| **总复杂度** | **O(nkL)**，接近线性 |

### 三种方案对比

```
n = 100 万条数据

MD5 精确去重:        O(n)    → 秒级，但只能找完全相同的
MinHash 暴力两两比较: O(n²)   → 不可能完成
MinHash + LSH:      O(n·k)  → 分钟级，k=128 时约是 MD5 的 128 倍
```

LSH 的价值：**把本来 O(n²) 不可能完成的模糊去重，压缩到接近线性的复杂度。**

---

## 三、如何选择

| 场景 | 推荐算法 |
|---|---|
| 网页爬取数据，存在大量完全重复页面 | `document_deduplicator`（快） |
| SFT 数据，同一问题被不同人改写成多个版本 | `document_minhash_deduplicator` |
| 数据量极大（>1亿条），对速度要求高 | `document_deduplicator` 先粗过，再 MinHash 细过 |

---

## 四、关键参数说明

`document_minhash_deduplicator` 的参数：

| 参数 | 说明 | 调整方向 |
|---|---|---|
| `tokenization` | 分词方式，中文用 `character`，英文用 `space` | 按语言选择 |
| `window_size` | n-gram 窗口大小 | 越大对局部相似越敏感 |
| `num_permutations` | 签名长度，越大估算越准 | 精度和速度的权衡 |
| `jaccard_threshold` | 相似度阈值，超过则视为重复 | 越小过滤越激进 |
| `num_bands` | LSH 的 band 数量，默认自动计算最优值 | 一般不需要手动设置 |
